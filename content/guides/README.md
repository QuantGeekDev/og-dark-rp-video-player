# Guides

Short player-facing markdown guides for the in-game Tutorials section.

Content rules:

- Keep each guide concise enough for in-game UI.
- Use `## Quick Start`, `## Details`, and `## Gotchas` sections.
- Prefer short bullets over tables.
- Keep exact exploit-prone implementation details in developer docs, not player guides.
- Update the corresponding implementation plan when a major mechanic changes.
